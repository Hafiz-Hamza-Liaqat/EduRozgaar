# Documentation archive

Historical sprint reports, audits, launch checklists, and QA artifacts.

**Active production docs** remain in `docs/` root (deployment, security, operations, architecture contracts, current RC reports).

| Folder | Contents |
| ------ | -------- |
| `sprints/` | Sprint A–C implementation reports |
| `l2/` | L.1 / L.2 audits and sprint reports |
| `launch/` | Launch completion / QA checklists |
| `historical/` | Older QA, i18n, responsive, cleanup plans |
| `audits/` | Pre-launch admin audits |
| `qa/` | Historical QA scripts/artifacts |

Do not treat archived docs as current runbooks.
